# 📋 Гайд по оформлению Базы Знаний

## 1. Структура файла (Стандарт)
Каждый файл должен начинаться с YAML-блока. **Важно:** чтобы Obsidian распознавал свойства (Properties), ключи должны быть на латинице (type, status, tags, source), а значения можно писать на русском.

```markdown
---
type: концепт | новости | синтез | материалы
status: черновик | исследование | база
source: [[...]]
tags: [область/маркетинг, стадия/исследование]
---
```

## 2. Система тегов (Для Графа)
Используем иерархию через слэш `/`, но в YAML-блоке пишем их **БЕЗ символа #** (Obsidian добавит его сам).

### Области (область/...)
- `область/копирайтинг` — все про тексты.
- `область/ии_инструменты` — обзоры моделей.
- `область/маркетинг` — общие стратегии.

### Стадии работы (стадия/...)
- `стадия/исследование` — сбор фактов и поиск.
- `стадия/кристаллизация` — формирование концепта.
- `стадия/синтез` — готовая болванка для нейронки.

## 3. Визуализация в Графе
- **Цвет стадий:** Настрой в Obsidian (Graph Settings -> Groups) цвет для тегов, например для `стадия/синтез`.
- **Связи:** Всегда делай [[ссылку]] на родительский концепт.
