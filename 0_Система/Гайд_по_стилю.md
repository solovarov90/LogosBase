# 📋 Гайд по оформлению Базы Знаний

## 1. Структура файла (Стандарт)
Каждый файл должен начинаться с YAML-блока. Ключи на латинице, теги записываем явно с символом `#` в кавычках (для стабильности Obsidian).

```markdown
---
type: концепт | новости | синтез | материалы
status: черновик | исследование | база
source: [[...]]
tags: 
  - "#область/маркетинг"
  - "#стадия/исследование"
---
```

## 2. Система тегов (Для Графа)
Используем иерархию через слэш `/`. В YAML пишем явно: `#область/маркетинг`.

### Области (#область/...)
- `#область/копирайтинг`
- `#область/ии_инструменты`
- `#область/маркетинг`

### Стадии работы (#стадия/...)
- `#стадия/исследование`
- `#стадия/кристаллизация`
- `#стадия/синтез`

## 3. Настройка групп в Графе (Скопируй в Obsidian)
Чтобы граф был красивым, нажми на значок шестеренки в Графе -> **Groups** -> **New Group** и вставь:

1. **Зеленый (Готовый контент):** `tag:#стадия/синтез`
2. **Синий (Ядро знаний):** `tag:#стадия/кристаллизация`
3. **Желтый (Сырые данные):** `tag:#стадия/исследование`
4. **Красный (Психология/Страхи):** `tag:#тема/психология`
