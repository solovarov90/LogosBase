# 📋 Гайд по оформлению Базы Знаний

## 1. Структура файла (Стандарт)
Каждый файл должен начинаться с YAML-блока (для системного поиска) и содержать блок параметров (для ИИ).

```markdown
---
тип: концепт | новости | синтез
статус: черновик | развитие | база
источник: [[...]]
теги: [#область/..., #стадия/...]
---
# [Название]

> [!abstract] Параметры для ИИ
> **Цель:** 
> **Фактура:** 
> **Контекст:**
```

## 2. Система тегов (Для Графа)
Используем иерархию через слэш `/`, чтобы в Obsidian можно было сворачивать ветки.

### Области (#область/...)
- `#область/копирайтинг` — все про тексты.
- `#область/ии_инструменты` — обзоры моделей.
- `#область/маркетинг` — общие стратегии.

### Стадии работы (#стадия/...)
- `#стадия/исследование` — сбор фактов и поиск.
- `#стадия/кристаллизация` — формирование концепта.
- `#стадия/синтез` — готовая болванка для нейронки.

### Темы (#тема/...)
- `#тема/промптинг`
- `#тема/психология`
- `#тема/автоматизация`

## 3. Визуализация в Графе
- **Цвет стадий:** Настрой в Obsidian (Graph Settings -> Groups) цвет для `#стадия/синтез` (например, зеленый), чтобы видеть готовые к работе файлы.
- **Связи:** Всегда делай [[ссылку]] на родительский концепт из файла синтеза.
